# Platform and Network Summary

Основано прежде всего на:
- Andrew Chen — `The Cold Start Problem`
- Parker, Van Alstyne, Choudary — `Platform Revolution`

## Зачем это важно

Эта рамка нужна, когда продукт создает ценность не только сам по себе, а через взаимодействия между участниками.

## Главная мысль

Платформа становится сильной не тогда, когда у нее много пользователей, а тогда, когда она запускает устойчивый ценный обмен между сторонами.

## Ключевые идеи

### 1. Atomic network

Нужно определить минимальную самодостаточную сеть:
- кто участники
- какой между ними exchange
- в какой момент продукт уже полезен сам по себе

### 2. Cold start is structural

Проблема старта сети не решается только маркетингом.
Нужно сознательно спроектировать:
- где взять первых правильных участников
- как создать первую плотность взаимодействий
- как дать value до масштаба

### 3. Liquidity beats vanity growth

Важно не общее число участников, а вероятность быстро получить нужное взаимодействие.

### 4. Retention loops

Сеть становится сильной, когда взаимодействия формируют причину вернуться, а не просто зарегистрироваться.

### 5. Governance

Платформа масштабирует не только ценность, но и мусор.
Поэтому правила, модерация и качество взаимодействий нужно проектировать рано.

## Практический вывод для продукта

Спрашивай:
- что является atomic network
- где главный cold start bottleneck
- как выглядит liquidity
- что создает retention loop
- какая governance-механика обязательна с самого начала

## Ошибка, которую эти книги помогают избегать

Не путай audience growth с network strength.
