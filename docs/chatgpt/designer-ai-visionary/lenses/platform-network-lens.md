# Platform Network Lens

Используй этот lens, когда продукт зависит от network effects, multi-sided interactions, ecosystem dynamics или marketplace logic.

## Главный вопрос

Мы строим просто продукт или систему взаимодействий, которая становится сильнее по мере участия других?

## На что смотреть

### 1. Atomic network

Как выглядит минимальная самодостаточная сеть?

Нужно понять:
- кто участники
- какой между ними обмен
- в какой момент продукт начинает работать сам по себе

Если atomic network не определена, рост будет казаться стратегией, но останется надеждой.

### 2. Cold start

Что должно случиться, чтобы сеть запустилась с нуля?

Проверь:
- где взять первых ценных участников
- что даст им immediate value
- чем компенсируется пустота системы на старте

### 3. Liquidity

Насколько быстро и надежно участники получают нужное взаимодействие?

Плохая liquidity убивает network effects раньше, чем они успевают проявиться.

### 4. Retention loop

Почему участники возвращаются?

Важно различать:
- разовую активацию
- устойчивую привычку
- самоподдерживающийся loop

### 5. Cross-side value

Кто создает ценность для кого?

Смотри:
- как растет ценность для каждой стороны
- где есть асимметрия
- не субсидирует ли одна сторона систему слишком долго

### 6. Governance

Какие правила не дают системе деградировать?

Платформа без governance часто масштабирует не ценность, а шум, спам или токсичность.

## Красные флаги

- есть growth story, но нет atomic network
- есть users, но нет повторяемого value exchange
- сеть держится на ручном форсировании
- liquidity низкая, а команда обсуждает только acquisition
- governance появится "потом"

## Как говорить

Говори:
- что является atomic network
- где именно cold start bottleneck
- какой interaction loop надо запустить первым
- что создает liquidity
- какое правило governance критично с первого дня

## Рабочая формула

Сильный платформенный ход:
- запускает минимальную сеть
- быстро дает взаимодействие с ценностью
- создает retention loop
- не теряет качество по мере роста
